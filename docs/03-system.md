# システム定義

## 1. プロダクト定義

本システムはAI文章生成ツールではない。営業の思考を構造化するフレームワークを実装したツールである。「仮説を書くAI」ではなく、「仮説を構造化するフレームワークを動かすツール」とする。

### 設計原則

- 仮説の質を標準化するツールである。出力の速さや量より、仮説の型と質の再現性を優先する。
- とにかく早くするのは避ける。質を損なってまで速度を追求しない。クロール・LLMの設計でも、中身を削って短時間に寄せるより仮説の質を守る方を選ぶ。

## 2. コアバリュー

企業ごとの営業仮説（hypothesis）と提案文下書き（letter_draft）を生成することを目的とする。「企業を探す」「情報を集める」は手段であり、中心価値ではない。中心価値は営業仮説の構造化と再現性である。

## 3. アウトプット

- メイン：hypothesis（営業仮説）、letter_draft（提案文下書き・編集可能）
- サブ：summary_business（事業要約）。経営課題の可視化が目的ではなく、仮説構築の材料とする。MVPではIR要約は扱わない。

## 4. 入力（MVP）

企業URL入力とHP情報取得のみとする。企業探索は目的ではない。

## 5. ターゲット

日本企業向け営業とし、受託型ビジネスを行う企業を主な顧客とする。初期市場は中小向け受託営業とする。

## 6. MVP方針

安全で狭いMVPから開始する。コアは仮説生成と提案文下書き生成とする。不要機能は削る前提とする。

## 7. ポジショニング

営業思考フレームワーク実装ツールとして売る。AI文章生成市場では戦わない。
