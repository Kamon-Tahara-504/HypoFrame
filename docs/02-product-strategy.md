# プロダクト戦略

## 1. 背景

営業活動における最大の非効率は、企業理解と仮説構築が属人化していることである。多くの営業は企業情報収集に時間がかかり、仮説の作り方が体系化されておらず、提案文作成がゼロベースになっている。このため、AIで文章を書くのではなく、営業の思考プロセス自体を構造化する方向を採用した。

## 2. ポジション

AI文章生成ツール・企業検索ツール・経営課題分析ツールは採用しない。**営業思考フレームワーク実装ツール**として定義する。文章生成市場は競合が多く差別化が困難で価格競争に陥りやすい一方、「営業の思考を構造化する」領域はまだ明確な競合が少ないためである。

## 3. ブランド

- ブランド：営業の思考を構造化するツール
- 思想：属人化した営業仮説構築を、再現可能なプロセスへ変える

### ツール設計の原則

- 仮説の質を標準化するツールである。速さ・件数より質を優先する。
- とにかく早くするのは避ける。質を犠牲にした速度優先はしない。

## 4. 長期ビジョン

単なるツールではなく、営業フレームワークの実装OSへ進化させる可能性を持つ。思考の標準化、組織での再現性、若手育成への応用を想定する。

## 5. 制作目的（現時点）

本プロジェクトの制作目的は、現時点では**学習・自分自身の成果物**に近い。顧客・契約の記述は、受託で使う場合や「どこまで考えて制作したか」を残すために含めている。売り方や契約形態は後から変えられるが、考えの深さを設計に残すことを重視する。販売形態は受託営業会社向けの専用武器または営業教育（思想販売）に展開可能とする。MVP 段階では学習・成果物として制作し、受託利用時は契約でスコープを定める。

## 6. 制約

予算はできるだけ無料で実装する。日本企業向け設計とする。受託販売で使う場合は契約で定める。
