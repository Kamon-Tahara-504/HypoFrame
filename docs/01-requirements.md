# 要件定義

## 1. プロダクト目的

中小企業向け受託営業において、営業仮説の構造化を実現する。文章生成ではなく、仮説思考の再現性を提供する。

## 2. スコープ（MVP）

### 必須機能

1. 企業URL入力
2. Web情報取得（HPのみ）
3. 事業要約生成
4. 固定構造の営業仮説生成
5. 仮説に基づく提案文下書き生成
6. 仮説・提案文の編集機能
7. テキストエクスポート

### 除外機能（MVPでは行わない）

企業検索機能、IR解析、業界別テンプレ分岐、CRM連携、メール送信機能、スコアリング機能、多言語対応、チーム管理機能。

## 3. 仮説構造（固定）

仮説は必ず次の5段・この順序で出力する。

1. 企業の現在状況整理
2. 潜在課題の仮説
3. 課題の背景要因
4. 改善機会（介入ポイント）
5. 提案仮説

出力は常にこの順序を守る。

## 4. ユーザー想定

対象は中小企業向け受託営業を行う企業とする。利用者は営業担当者・営業責任者・少人数組織とする。

## 5. 非機能要件

- 無料で構築可能な技術選定とする。
- APIコストを最小化する。
- 情報取得失敗時は安全に処理する。
- 「仮説である」ことを画面で明確に表示する。
- 設計原則：仮説の質を標準化するツールであり、とにかく早くするのは避ける（質を損なってまで速度を優先しない）。

## 6. 成功指標（初期）

1社あたりの仮説生成時間短縮、営業担当者の満足度、仮説構造の再現性を測る。

## 7. リスク

仮説精度への過度な期待、ChatGPTとの差別化不明瞭、中小企業HPの情報不足を認識している。

## 8. 現フェーズ目標

思想をブレさせず、最小構成で動くプロダクトを作る。
